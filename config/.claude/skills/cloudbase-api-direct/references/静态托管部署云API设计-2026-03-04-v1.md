## 一、部署管理

### 1. 查询静态托管部署列表（DescribeStaticDeployList）

#### 1.1 接口描述

查询指定环境下的静态托管部署列表，支持分页、搜索和服务名称过滤。

#### 1.2 输入参数

| 参数名称 | 必选 | 类型 | 描述 |
| :--- | :--- | :--- | :--- |
| Action | 是 | String | 公共参数，本接口取值：`DescribeStaticDeployList` |
| Version | 是 | String | 公共参数，本接口取值：`2018-06-08` |
| Region | 否 | String | 公共参数，本接口不需要传递此参数 |
| EnvId | 是 | String | 环境ID<br>示例值：`env-abc123` |
| Uin | 是 | String | 用户UIN<br>示例值：`100012345678` |
| Business | 是 | String | 业务类型<br>示例值：`webify` |
| PageNo | 否 | Integer | 页码，从1开始，默认为1<br>示例值：`1` |
| PageSize | 否 | Integer | 每页数量，默认10，最大100<br>示例值：`10` |
| SearchKey | 否 | String | 搜索关键字，支持按服务名模糊搜索<br>示例值：`my-app` |
| Filter | 否 | ServiceListFilter | 过滤条件 |

| :--- | :--- | :--- |

### 2. 创建静态托管部署（CreateStaticDeploy）

#### 2.1 接口描述

创建静态托管部署服务并触发构建。支持多种代码源类型（GIT、ZIP、TEMPLATE、BASE64）。创建成功后将异步执行构建任务，可通过版本查询接口获取构建状态。

#### 2.2 输入参数

| 参数名称 | 必选 | 类型 | 描述 |
| :--- | :--- | :--- | :--- |
| Action | 是 | String | 公共参数，本接口取值：`CreateStaticDeploy` |
| Version | 是 | String | 公共参数，本接口取值：`2018-06-08` |
| Region | 否 | String | 公共参数，本接口不需要传递此参数 |
| EnvId | 是 | String | 环境ID<br>示例值：`env-abc123` |
| Uin | 是 | String | 用户UIN<br>示例值：`100012345678` |
| Business | 是 | String | 业务类型<br>示例值：`webify` |
| ServiceName | 是 | String | 部署服务名称<br>示例值：`my-app` |
| Config | 是 | String | 部署配置，JSON字符串格式。包含构建类型、框架、代码源等配置信息。详见下方 Config JSON 说明<br>示例值：见下方 |
| BuildType | 否 | String | 构建类型：`GIT`-Git仓库构建、`ZIP`-ZIP包构建、`TEMPLATE`-模板构建、`BASE64`-Base64编码构建<br>示例值：`GIT` |
| Domain | 否 | String | 域名<br>示例值：`my-app.example.com` |

**Config JSON 结构说明：**

| 字段名称 | 类型 | 描述 |
| :--- | :--- | :--- |
| BuildType | String | 构建类型：GIT / ZIP / TEMPLATE / BASE64 |
| Framework | String | 框架类型，如：vue、react、nextjs 等 |
| Runtime | String | 运行时类型，如：nodejs、python |
| NodeJsVersion | String | Node.js 版本，默认 `20` |
| PythonVersion | String | Python 版本，默认 `3.10` |
| AppPath | String | 应用路径（构建输出目录） |
| BuildPath | String | 构建路径（源码子目录） |
| ZipFileUrl | String | ZIP文件地址（BuildType 为 ZIP/TEMPLATE/BASE64 时使用） |
| CodeSource | String | 代码源平台（BuildType 为 GIT 时使用），如：github、gitlab、coding、gitee |
| CodeRepo | String | 代码仓库名（BuildType 为 GIT 时使用） |
| CodeBranch | String | 代码分支（BuildType 为 GIT 时使用） |
| UseDockerfile | Boolean | 是否使用 Dockerfile 构建 |
| DockerfileDir | String | Dockerfile 所在目录 |
| DockerfilePath | String | Dockerfile 文件路径 |
| EnableEntInst | Boolean | 是否启用企业版 TCR 实例 |
| TcrId | String | TCR 实例ID |
| TcrRegion | String | TCR 实例地域 |
| UseScfTempCos | Boolean | 是否使用 SCF 临时 COS |
| BuildParams | Object | 构建参数，键值对格式。支持 INSTALL_CMD、BUILD_CMD、DEPLOY_CMD |
| BuildEnv | Object | 构建环境变量，键值对格式 |
| CosTimestamp | String | COS 时间戳（BuildType 为 ZIP 时使用） |
| CosSuffix | String | COS 文件后缀（BuildType 为 ZIP 时使用） |

#### 2.3 输出参数

| 参数名称 | 类型 | 描述 |
| :--- | :--- | :--- |
| Data | CreateStaticDeployResp | 结果返回 |
| RequestId | String | 唯一请求 ID，由服务端生成，每次请求都会返回 |

**CreateStaticDeployResp 结构：**

| 参数名称 | 类型 | 描述 |
| :--- | :--- | :--- |
| BuildId | String | 构建ID |
| VersionName | String | 自动生成的版本名称（格式：{ServiceName}-{序号}，如 `my-app-001`） |

#### 2.4 示例

**示例1：使用 Git 仓库创建部署**

**输入示例：**

```http

}
```

**输出示例：**

```json

        "Data": {
            "BuildId": "2600000001",
            "VersionName": "my-vue-app-001"
        },
        "RequestId": "76c0ca0f-4cf2-4f89-80b9-cdbf9ab7dd77"
    }
```

```json

        "Data": {
            "BuildId": "2600000002",
```

#### 2.5 错误码

| 错误码 | 描述 |
| :--- | :--- |
| AuthFailure | CAM签名/鉴权错误 |
| FailedOperation | 操作失败 |
| FailedOperation.DuplicatedData | 数据重复 |
| InternalError | 内部错误 |
| InvalidParameter | 参数错误 |
| InvalidParameterValue | 参数取值错误（如 Config JSON 格式不正确） |
| ResourceNotFound | 资源不存在 |

#### 2.6 接口说明

1. **版本号自动递增**：每次创建部署会自动生成版本号，格式为 `{ServiceName}-{序号}`（如 `my-app-001`、`my-app-002`），序号至少3位，超过999时自动扩展位数。
2. **异步构建**：创建成功后构建任务异步执行，通过 `DescribeStaticDeployVersion` 或 `DescribeStaticDeployInfo` 查询最新构建状态。
3. **状态轮询**：后端每10秒自动轮询构建状态，超时时间为10分钟，超时后状态标记为 `FAILED`。
4. **首次创建**：如果部署服务不存在，接口会自动创建服务记录后再触发构建。

| Uin | 是 | String | 用户UIN<br>示例值：`100012345678` |
| Business | 是 | String | 业务类型<br>示例值：`webify` |
| ServiceName | 是 | String | 部署服务名称<br>示例值：`my-app` |

#### 4.3 输出参数

| 参数名称 | 类型 | 描述 |
| :--- | :--- | :--- |
| Data | DeleteStaticDeployResp | 结果返回 |

| 参数名称 | 类型 | 描述 |

```json
{
    "Response": {
        "Data": {
            "Result": true
        },
        "RequestId": "d4e5f6a7-b8c9-0123-def0-234567890123"
    }
```

#### 4.5 错误码

| 错误码 | 描述 |
| :--- | :--- |
| AuthFailure | CAM签名/鉴权错误 |
| FailedOperation | 操作失败（删除版本或服务记录失败） |

2. **TCR 清理**：如果最新版本使用了 Dockerfile 构建（UseDockerfile=true），将异步删除对应的 TCR 镜像仓库，不影响接口响应速度。

### 8. 获取静态托管部署 COS 信息（DescribeStaticDeployCosInfo）

#### 8.1 接口描述

获取静态托管部署的 COS 文件上传/下载链接。用于 ZIP 包构建场景下的文件上传和下载。

#### 8.2 输入参数

| 参数名称 | 必选 | 类型 | 描述 |
| :--- | :--- | :--- | :--- |
| Action | 是 | String | 公共参数，本接口取值：`DescribeStaticDeployCosInfo` |
| Version | 是 | String | 公共参数，本接口取值：`2018-06-08` |
| Region | 否 | String | 公共参数，本接口不需要传递此参数 |
| EnvId | 是 | String | 环境ID<br>示例值：`env-abc123` |
| Uin | 是 | String | 用户UIN<br>示例值：`100012345678` |
| Business | 是 | String | 业务类型<br>示例值：`webify` |
| ServiceName | 是 | String | 部署服务名称<br>示例值：`my-app` |
| UnixTimestamp | 否 | String | Unix时间戳，不传则自动生成当前时间戳。用于生成 COS 存储 Key<br>示例值：`1709500000` |
| Suffix | 否 | String | 文件后缀，默认 `.zip`。如不以 `.` 开头将自动补充<br>示例值：`.zip` |
| NeedDownload | 否 | Boolean | 是否获取下载链接。`true`-获取下载链接，`false`-获取上传链接（默认）<br>示例值：`false` |

#### 8.6 接口说明

1. **上传流程**：先调用此接口获取上传链接（NeedDownload=false），然后使用返回的 `UploadUrl` 上传文件，最后调用 `CreateStaticDeploy` 触发构建。
2. **下载流程**：传入与上传时相同的 `UnixTimestamp` 和 `Suffix`，设置 NeedDownload=true 获取下载链接。
3. **Key 生成规则**：COS Key 格式为 `{Business前6位}-{EnvId后6位}-{ServiceName前8位}-{Timestamp前10位}`，由系统自动生成。

## 七、典型使用流程

### 流程1：通过 Git 仓库部署

```
1. CreateStaticDeploy（BuildType=GIT）   → 获取 BuildId、VersionName
2. DescribeStaticDeployVersion           → 轮询构建状态直到 SUCCESS/FAILED
3. DescribeStaticDeployInfo              → 确认部署信息
```

### 流程2：通过 ZIP 包部署

```
1. DescribeStaticDeployCosInfo（NeedDownload=false） → 获取上传链接和 UnixTimestamp
2. 使用 UploadUrl 上传 ZIP 文件
3. CreateStaticDeploy（BuildType=ZIP，Config 中传入 CosTimestamp） → 获取 BuildId、VersionName
4. DescribeStaticDeployVersion → 轮询构建状态直到 SUCCESS/FAILED
```

### 流程3：查看和管理部署

```
1. DescribeStaticDeployList              → 查看环境下所有部署
2. DescribeStaticDeployVersionList       → 查看某个部署的所有版本
3. DeleteStaticDeployVersion             → 删除某个版本
4. DeleteStaticDeploy                    → 删除整个部署服务
```
