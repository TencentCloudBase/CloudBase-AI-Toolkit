# CloudBase AI 插件

一键把 CloudBase 接入 AI 编程助手：同时捆绑 [MCP Server](/ai/cloudbase-ai-toolkit/getting-started) 与 [Agent Skills](/ai/cloudbase-ai-toolkit/prompts/how-to-use)，安装后即可查库、管云函数、部署托管等，无需再分别配置。

相比单独装 Skills，插件还带上 Hooks 等能力，**Skill 激活更稳定**，推荐优先用插件。

## 安装

### 方式 1：一键安装（推荐）

通过兼容 [Open Plugin Specification](https://open-plugins.com/plugin-builders/specification) 的 `plugins` CLI 安装，会自动检测本机 AI 工具：

```bash
# CloudBase 主插件（MCP + Agent Skills）
npx plugins add TencentCloudBase/cloudbase-plugin

# CloudBase Sites 插件（Vite Web 创建与部署）
npx plugins add TencentCloudBase/cloudbase-sites-plugin
```

加 `--yes` 可跳过确认。指定目标或范围：

```bash
npx plugins add TencentCloudBase/cloudbase-plugin --target cursor
npx plugins add TencentCloudBase/cloudbase-plugin --scope local
```

**自动检测支持：** Claude Code、Cursor、Codex、Grok Build、Kimi Code、GitHub Copilot CLI、VS Code（Preview）。安装后重启对应工具即可。

> Claude Code / Codex 的原生 marketplace 请用主仓库 `TencentCloudBase/CloudBase-MCP`；`cloudbase-plugin` 仓库由 CI 同步 OPS 标准内容，不含 marketplace 元数据。

> ⚠️ 已通过方式 2（`claude plugin install` / `codex plugin add` 等）安装过时，**不要**再对同一 IDE 执行 `npx plugins add`，以免重复实例。二选一即可。

### 方式 2：按 IDE 手动安装

适用于 CodeBuddy、WorkBuddy、ZCode，或希望走 IDE 原生插件机制的场景：

<Tabs>
  <TabItem value="codex-app" label="Codex App">

1. 在 Codex App 右上角点 `+` → **添加插件市场**
2. 来源填 `TencentCloudBase/CloudBase-MCP`，Git 引用 `main`，稀疏路径留空
3. 在 **Tencent CloudBase** 市场中安装 `cloudbase`
4. 新开线程验证

  </TabItem>
  <TabItem value="codex-cli" label="Codex CLI">

```bash
codex plugin marketplace add TencentCloudBase/CloudBase-MCP --ref main
codex plugin add cloudbase@tencent-cloudbase
```

新开线程后可用 `codex plugin list` 确认。

  </TabItem>
  <TabItem value="claude-code" label="Claude Code">

```bash
claude plugin marketplace add TencentCloudBase/CloudBase-MCP
claude plugin install cloudbase@tencent-cloudbase
```

安装后执行 `/reload-plugins`。默认 `--scope user`；团队共享可用 `--scope project`。

  </TabItem>
  <TabItem value="workbuddy" label="WorkBuddy">

1. 对话输入框左下角点 **连接器**
2. 找到 **腾讯云 CloudBase** → **连接**

也可输入 `/cloudbase` 选择 Skill。详见 [WorkBuddy 配置指南](ide-setup/workbuddy)。

![WorkBuddy 内置 CloudBase 连接器](https://7463-tcb-advanced-a656fc-1257967285.tcb.qcloud.la/assets/workbuddy-connector-full.png)

  </TabItem>
  <TabItem value="zcode" label="ZCode">

ZCode（≥ 3.4.1）设置 → **插件** → **开发者工具** → 启用 **cloudbase-skills**。

也可在 **MCP 服务器** 中手动添加：命令 `npx`，参数 `-y @cloudbase/cloudbase-mcp@latest`。详见 [ZCode 配置指南](ide-setup/zcode)。

  </TabItem>
</Tabs>

### 其他 IDE

不走 marketplace 的工具（Cursor、WindSurf、CodeBuddy 等）可配置 MCP + Skills。选择你的工具查看步骤：

<IDESelector defaultIDE="cursor" showInstallButton={false} />

> CodeBuddy / WorkBuddy / ZCode 尚未实现 Open Plugin Specification，因此暂不支持 `npx plugins` 一键安装，需用各自原生路径。

## 插件里有什么

| 能力 | 说明 |
|------|------|
| **MCP Server** | 环境、数据库、云函数、托管、存储、云托管、认证、权限、日志等，详见 [MCP 工具](mcp-tools.md) |
| **Agent Skills** | 平台、Web / 小程序、云函数、认证、数据库、AI 模型、UI 等专业知识，详见 [如何使用 Skill](prompts/how-to-use.mdx) |
